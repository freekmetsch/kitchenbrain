Architecture plan · R2 · four phases

One agent. Every screen. Explicit context.

A root-owned conversation opened from a draggable edge bubble. Pages publish small semantic facts; the server validates and injects them for one model turn. No DOM scraping, context persistence, new dependency, or authorization shortcut.

Authority and data flow

Page publishes whitelisted ScreenContextV1
Root controller snapshots context on Send
Chat API validates shape and size
Model sees ephemeral untrusted context
Existing tools re-read SQLite and enforce confirmations

Options considered

Rejected

Wrap current ChatView globally

Looks small, but duplicates local state, persists machine context visibly, and cements an 839-line mixed controller/view component.

Chosen

Root controller + typed context

One transcript across Home and the agent panel, explicit privacy boundary, existing tools and drag package retained.

Later, selectively

Server route/entity resolver

Strong authority but duplicates existing tool reads and cannot express transient filters or dirty edit mode without a resolver per route.

Two contracts make it safe

ScreenContextV1

  • Route label, entity identity, at most 12 simple facts.
  • Approximately 4 KiB hard limit; unknown fields rejected.
  • No DOM text, secrets, settings values, notes, or draft contents.
  • Dirty edit state blocks writes to the entity.
  • Never persisted to chat history.

ChatAgentController

  • One layout-instantiated controller; never an SSR module singleton.
  • Owns transcript, draft, stream, attachments, confirmations, Undo.
  • Home and floating panel are renderers, not separate chats.
  • Snapshot on Send prevents navigation from mutating an active turn.
  • Lazy history hydration runs once.

Execution phases

Extract without changing behavior

Characterize current chat, extract a root-safe controller, and hydrate lazily. Do not build the bubble first.

Add the ephemeral context seam

Strict schema, JSON/multipart parsing, model-only injection, system rule, and malformed/forged request tests.

Mount the bubble and full chat

Draggable edge bubble; anchored desktop panel; full-viewport mobile dialog; horizontal containment; coordinated layers and focus.

Publish route context and replace recipe handoffs

Rich context on core routes, safe label fallback elsewhere, in-place role prompt, batched page invalidation, English/Dutch copy.

Failure modes designed out

SSR user leakageLayout instance, never module singleton; concurrent SSR test.
Duplicate transcriptsOne controller and idempotent hydrateOnce.
Prompt injection / secretsTyped facts, no scraping, bounded size, untrusted marker, authoritative tool rereads.
Unsaved-form racesDirty state publishes identity only and blocks entity writes.
Navigation kills stream/filesController owns abort and object-URL lifecycle until root destruction.
Overlay and drag trapsAgent layer token, modal mobile surface, click/keyboard alternatives, Escape and return focus.
Refresh stormsBatch one invalidate after tool stream; immediate only for Undo/approval.
Token/cost creep4 KiB/12-fact cap and existing spend accounting.
